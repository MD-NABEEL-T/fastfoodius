🔥 The Most Important Shift

Right now your system does:

Student → Order → Stored in DB → Then payment verification happens manually.

That is unsafe in production.

Your new architecture says:

Payment must be verified first
Then stock updated
Then order created

That is correct.

🧠 Correct Final Flow (Explained Clearly)
1️⃣ Student logs in

Firebase Auth — already done ✅

Optional upgrade later:
Email verification enforcement.

2️⃣ Student selects items

Frontend calculates total.

No order is stored yet.

3️⃣ Frontend calls Backend

Backend (Cloud Function) creates Razorpay order.

Why backend?

Because Razorpay secret key must NEVER be exposed.

Frontend receives:

razorpay_order_id

amount

4️⃣ Razorpay Checkout Opens

User pays.

If payment fails → nothing happens.

If payment succeeds → Razorpay sends:

payment_id

order_id

signature

To frontend.

5️⃣ VERY IMPORTANT STEP

Frontend sends:

payment_id
order_id
signature
cart details

To Cloud Function.

6️⃣ Cloud Function Verifies Signature

Using Razorpay secret key.

If invalid → reject.

If valid → continue.

This prevents fake payments.

7️⃣ Atomic Stock Transaction

This must be done server-side.

For each item:

Check:

Current stock

Requested quantity

If stock insufficient:
→ Trigger refund via Razorpay API
→ Stop process

If stock sufficient:
→ Deduct stock
→ Continue

This must be atomic (transaction block).

8️⃣ Generate Daily Token

Example:

2026-02-28:

Token numbers:
1, 2, 3, 4, 5...

Reset daily.

9️⃣ Create Order in Database

Now and ONLY now:

Store:

{
  token,
  items,
  total,
  payment_id,
  razorpay_order_id,
  payment_timestamp,
  status: "Pending",
  paymentStatus: "Verified"
}

Now owner sees it.

🔟 Owner Updates Status

Pending → Prepared → Served

That part you already built well.
